## 1. 전체 흐름 개요

```
시작
 └─ [Step 1] 글을 쓰는 목적이 무엇인가요?
      ├─ 부업으로 수익    → 웹소설 경로
      ├─ 취미/자기만족/자기계발 → 글 형식 선택
      └─ 잘 모르겠어요    → 🤖 장르 한 줄 설명 → 선택
```

> ⬅️ **뒤로가기** 및 ⏭️ **나중에 결정할게요** 버튼은 모든 단계에서 노출

## 2. Step 1 — 글을 쓰는 목적이 무엇인가요?

**선택지 3가지**:

| 선택 | 다음 단계 |
|------|----------|
| 부업으로 수익을 내고 싶어요 | → 웹소설 경로 바로 진입 |
| 취미로 / 자기 만족 / 자기 계발 | → 글 형식 선택 화면 |
| 잘 모르겠어요 | → 🤖 장르별 한 줄 설명 제공 후 선택 |

---

### 분기 A. 잘 모르겠어요 선택 시
> **🤖 AI 역할**: 각 글쓰기 형식을 짧게 한 줄씩 설명해줌

- 설명 후 다시 선택지로 돌아와 목적 선택

---

### 분기 B. 취미/자기계발 선택 시 — 글 형식 선택

**선택지**:

| 형식 | 설명 |
|------|------|
| 웹소설 | 온라인 연재 소설 |
| 순문학 / 추리소설 / 동화 / 연극 / 시나리오 | 문학 장르 |
| 일기 / 에세이 | 생활 글쓰기 |

> ⚠️ 웹소설 외 형식의 세부 경로는 추후 설계 예정

---

### 분기 C. 부업 선택 시 / 웹소설 선택 시
→ 아래 **Step 2. 웹소설 경로**로 진입

---

## 3. Step 2 — 웹소설 경로: 장르 선택

**질문**: "생각해둔 장르가 있나요?"

**선택지**:

| 선택 | 다음 단계 |
|------|----------|
| 남성향 (판타지 / 무협 / 현판) | → 해당 장르로 확정 |
| 여성향 (로판 / 로맨스 / BL) | → 해당 장르로 확정 |
| 모르겠어요 | → 웹소설 첫 경험 여부 확인 |

---

### 모르겠어요 선택 시

**질문**: "웹소설이 처음이신가요? 간단하게 설명해드릴까요?"

- **아니오** → 장르 선택지 다시 표시
- **네** → 🤖 장르별 한 줄 설명 제공

#### 🤖 장르 설명 내용 (AI 또는 고정 텍스트로 제공)

**남성향**
| 장르 | 설명 |
|------|------|
| 판타지 / 퓨전 | 이세계 진출, 드래곤, 마법 등 |
| 현대 판타지 (현판) | 현대 배경의 헌터물, 재벌물, 전문가물 (의사, 변호사 등) |
| 무협 | 천마, 무림, 문파 간의 갈등을 다룬 무협 웹소설 |
| 게임 판타지 | 게임 속 세상에 들어가거나 게임 능력치를 가진 주인공의 성장기 |

**여성향**
| 장르 | 설명 |
|------|------|
| 로맨스 판타지 (로판) | 판타지 세계관에서의 로맨스. 공작, 황제, 계약 결혼 등의 소재 |
| 현대 로맨스 | 현대적인 배경의 남녀 간 사랑 이야기 |
| BL (Boys Love) | 남성 간의 로맨스를 다루는 장르 |

> 설명 후 → 장르 선택지로 돌아와 선택

## 4. Step 3 — 초안 작성

### Step 3-1. 초안이 있나요?

**질문**: "초안을 작성해 볼까요? 초안이 있나요?"

| 선택 | 다음 단계 |
|------|----------|
| YES (초안 있음) | → [기존 유저 경로] 초안 붙여넣기 후 수정 도움 |
| NO (초안 없음) | → Step 3-2로 진행 |

---

### Step 3-2. (초안 없음) 도움 제안

**질문**: "제가 도와드릴까요?"

| 선택 | 다음 단계 |
|------|----------|
| YES | → 🤖 뼈대 만들기 (4가지 질문) |
| NO | → 유저가 직접 작성 |

---

### Step 3-3. 🤖 뼈대 만들기 — 4가지 질문 (Gemini API 호출)

> **AI 역할**: 아래 4가지 질문에 대한 유저의 답변을 바탕으로 작품의 뼈대를 구성해줌.
> 각 질문에 AI가 힌트/예시를 제공하며 유저의 답변을 이끌어냄.

**질문 목록**:

| # | 질문 | 목적 |
|---|------|------|
| 1 | 이 작품의 장르와 소재가 주는 기대감이 무엇인가요? | 독자 훅(hook) 설정 |
| 2 | 주인공의 목표와 매력은 어떤 것인가요? | 캐릭터 설계 |
| 3 | 주인공의 성장 요소는 무엇이고, 그것이 특별하게 느껴지는 이유는 무엇인가요? | 서사 동력 설정 |
| 4 | 작품의 주제를 한 줄로 써주세요. | 핵심 메시지 정리 |

**질문 4 예시** (유저에게 함께 제공):
> "1차 세계대전 시대에 환생한 주인공이 미래 지식을 활용하여 미래를 바꿔서 승승장구하는 이야기"

---

### Step 3-4. 🤖 뼈대 기반 초안 생성 (Gemini API 호출)

> **AI 역할**: 4가지 질문 답변을 바탕으로 첫 장면 초안을 자동 생성.

- 입력값: 장르 + 4가지 뼈대 답변
- 출력값: 첫 장면 텍스트 초안
- **스트리밍 권장**: `streamGenerateContent` 엔드포인트 사용 시 실시간 타이핑 효과

---

### Step 3-5. 🤖 수정/피드백 루프 (Gemini API 호출)

> **AI 역할**: 초안을 유저 요청에 따라 반복적으로 다듬어줌.

**수정 옵션**:
- 문장 다듬기
- 더 구체적으로
- 길이 조절
- 어조 변경
- 맞춤법/문법 교정

유저 만족 시 → **완료/내보내기**

## 8. 개발 참고사항

### AI 엔진
- **사용 모델**: `gemini-2.5-flash`
- **API 키 발급**: [Google AI Studio](https://aistudio.google.com)
- **엔드포인트**: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent`
- **스트리밍**: `streamGenerateContent` 사용 시 실시간 타이핑 효과

```js
// 기본 API 호출 구조
const response = await fetch(
  "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=YOUR_API_KEY",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      contents: conversationHistory,  // 전체 대화 히스토리
      systemInstruction: {
        parts: [{ text: systemPrompt }]  // 장르/목적 등 누적 컨텍스트
      }
    })
  }
)
```

### 상태 관리
- 유저 선택값(목적, 형식, 장르, 뼈대 답변 등)을 단계마다 누적 저장
- 수정 루프: 대화 히스토리 전체를 `contents` 배열로 매 호출마다 전달
- 뒤로가기 구현 시 각 단계의 상태를 스택으로 관리 권장

### 무료 한도 관리 ✅ 구현 완료
- localStorage 기기별 일일 카운팅 (`ai-usage-YYYY-MM-DD`)
- 200회 이상: AIPanel 헤더에 `{n}/250` 경고 뱃지 표시
- 250회 도달: 버튼 비활성화 + "오늘의 AI 도움 횟수를 모두 사용했어요" 배너
- 429 에러 서버 측: 분당 한도(RPM) 초과 → 최대 2회 exponential backoff (1.2s/2.4s)
- 429 에러 메시지 구분: 일별 한도 vs 분당 한도 친화적 메시지

### 미확정 항목
- 웹소설 외 형식(순문학, 에세이, 일기 등) 세부 경로 설계 필요
- 목적별(취미 vs 부업) 톤/방향 차별화 로직 미정

## 9. 성장 로드맵 (AI 비용 관점)

```
Phase 1 — MVP (무료 티어)
  └─ gemini-2.5-flash 무료 한도 내 운영
  └─ 하루 ~250회 AI 호출, 소규모 베타 유저 검증

Phase 2 — 유저 증가 시
  └─ Google AI Studio 결제 수단 등록 → pay-as-you-go
  └─ 예상 비용: 월 수만 원 수준 (소규모)

Phase 3 — 수익화
  └─ 무료 플랜: AI 기능 하루 N회 제한
  └─ 유료 구독 (월 정액): 무제한 AI 기능
  └─ 구독 수익으로 API 비용 충당 → 자립 구조
```
